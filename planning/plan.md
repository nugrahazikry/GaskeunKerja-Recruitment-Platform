## 1. What "Done" Means for the MVP

A person can, on a local machine, run the whole flow from the Direction B end-to-end spec:
1. HR logs in → posts a Job Description.
2. Candidate CVs get parsed → ranked against the JD (shortlist with explainable scores).
3. A ranked candidate takes an async AI interview (question → record → transcribe → rubric score).
4. HR sees the AI summary/score and makes the final decision (AI never auto-rejects).
5. Every candidate receives a development report (email or generated file).

No cloud hosting required — runs via local servers + Docker Compose. See `CLAUDE.md` for the local-substitute table.

## 6. Decisions — RESOLVED (2026-07-12)

- [x] **Structured DB** → **PostgreSQL (Docker)** via SQLAlchemy, no Alembic (`create_all`); SQLite is the ORM-level fallback.
- [x] **Matching** → **semantic vector similarity + lightweight competency-graph relations** (from the framework being curated). No porting of the un-integrated Tahap 2 KGE prototype; keeps the "KGE/relational" claim honest.
- [x] **Interview format** → **AUDIO voice recording is core** (revised 2026-07-12 Area-4 session; supersedes the earlier text-only call). MediaRecorder capture + Groq STT + audio storage all in scope.
- [x] **Speech-to-text** → **Groq `whisper-large-v3` API**, `language=id` (SumoPod has no STT). Local faster-whisper = drop-in fallback (RTX 3050).
- [x] **Recruiter question edit/approve** → **added** as a human-in-the-loop step between question-gen and candidate answer.
- [x] **LLM** → **all steps via SumoPod** (OpenAI-compatible, `https://ai.sumopod.com/v1`; `deepseek-v4-flash` / `deepseek-v4-pro`) + response caching; local-LLM substitution cut.
- [x] **Email** → **generate report as file + show/download in UI**; SMTP optional/mock (demo doesn't depend on delivery).
- [x] **Auth/RBAC** → **thin JWT + single role gate**; full RBAC matrix deferred.
- [x] **Reference datasets** → **scope to ONE demo job role** (~8-12 competencies, ~3 resources each); solo owns both the schema (code) and the curation (content).
- [x] **Tahap 2 reuse** → audit is Day-1 task (Area 2 T1 / Area 1 T1); CV-parse + FastAPI + React are the expected reuse anchors.
- [x] **Embeddings** → **`gemini/gemini-embedding-001` via SumoPod** (no separate API key — reuses `LLM_BASE_URL`/`LLM_API_KEY`), 1536-dim via Matryoshka truncation. **✅ Day-1 verified 2026-07-13**: model id requires the `gemini/` prefix (bare name → 400); `dimensions=1536` truncation confirmed (response vector is exactly 1536-dim). Local `paraphrase-multilingual-MiniLM-L12-v2` remains the documented fallback if needed later.
- [x] **Vision LLM** → **Groq `meta-llama/llama-4-scout-17b-16e-instruct` is PRIMARY** (not fallback). **✅ Day-1 verified 2026-07-13**: SumoPod's `deepseek-v4-pro` does **not** support vision — sent a test image, model's `reasoning_content` showed it reasoning "no image was provided," `prompt_tokens` far too low to have ingested image data, returned empty content. Groq's Llama 4 Scout correctly read text out of the same test image (`prompt_tokens: 174`, correct output). Reuses the existing Groq STT credentials, no new key needed.
